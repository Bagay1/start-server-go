# Go + Angular server setup script

Этот репозиторий содержит скрипт для быстрой настройки сервера под **Go-бэкенд** и **Angular-фронтенд**.  
Скрипт устанавливает и настраивает:

- PostgreSQL
- Nginx (прокси на Go + отдача статики Angular)
- systemd-сервис для Go-приложения
- пользователя, каталоги и переменные окружения

## Использование

1. **Запуск на чистом сервере Debian 12 (или Ubuntu)**
   ```bash
   curl -sSL https://raw.githubusercontent.com/Bagay1/start-server-go/refs/heads/main/deployGoSet.sh | sudo bash
   ```

   Во время выполнения будут заданы вопросы:
   - Имя приложения (по умолчанию `myapp`)
   - Домен (или оставить `_` для IP)
   - Email администратора (для будущего SSL)
   - Порт Go-приложения (по умолчанию `8080`)
   - Имя базы данных, пользователь, пароль

2. **После завершения скрипта**  
   - Будет создан каталог `/var/www/<имя_приложения>/`
   - В нём будут папки `bin`, `static`, `logs`, `config`
   - Сгенерирован файл `.env` с переменными окружения (включая `DATABASE_URL`)
   - Настроен systemd-сервис для автозапуска Go-приложения
   - Настроен Nginx, который проксирует `/api/` на Go-приложение и отдаёт статику из `/static`

3. **Развертывание приложения**  
   - Скомпилируйте ваш Go-бинарник для Linux:
     ```bash
     GOOS=linux GOARCH=amd64 go build -o myapp
     ```
   - Соберите Angular-приложение (папка `dist/` или `static/`).
   - Скопируйте бинарник и статику на сервер (например, в `/tmp`):
     ```bash
     scp myapp user@server:/tmp/
     scp -r dist/* user@server:/tmp/static/
     ```
   - На сервере выполните скрипт обновления:
     ```bash
     sudo -u myapp /var/www/myapp/update.sh
     ```
     (скрипт создаётся автоматически и копирует файлы в нужные места, затем перезапускает сервис)

4. **Проверка**  
   - Откройте в браузере `http://ваш_сервер/` – должна отобразиться Angular-страница.
   - API доступно по `http://ваш_сервер/api/...`

## Структура после установки

```
/var/www/myapp/
├── bin/
│   └── myapp               # Go-бинарник
├── static/                 # статика Angular (собранный проект)
├── logs/                   # логи (если приложение пишет туда)
├── config/
│   └── .env                # переменные окружения
├── update.sh               # скрипт для обновления бинарника/статики
```

## Настройка SSL (опционально)

После того как домен будет направлен на сервер, установите Certbot и получите сертификат:
```bash
apt install certbot python3-certbot-nginx -y
certbot --nginx -d ваш-домен
```

## Примечания

- Если вы не используете Angular, можете изменить конфигурацию Nginx, убрав блок `location /` или оставив только прокси.
- Скрипт не устанавливает Go на сервер – предполагается, что вы собираете бинарник локально.
- Пароль от PostgreSQL генерируется случайно и сохраняется в `.env`. Сохраните его в безопасном месте.
